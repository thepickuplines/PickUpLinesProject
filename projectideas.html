Project Idea

Introduction

The project that we will be making involves making a website that allows users to submit and rate pick-up lines submitted by other users, whether they are “good” or “bad” and what category of pick-up lines (e.g. sport, music, etc.). It will involve a database where user submissions and ratings for those submissions are stored with the appropriate category. Users should also be able to see the statistics for pick-up lines and also featured statistics that show the most popular lines in a data visualisation. The project shall be titled, "Rate My Pick-Up Line".

Description

“Rate My Pick-Up Line” is based on user-generated content (UGC); allowing users to create their pick-up line submissions to the website. Users can post anonymously or use an account to see their submissions (including statistics) and save pick-up lines made by other users.

When the user visits the website, the prominent feature that will be presented is a pick-up line on the screen, with a “good” button to indicate that the pick-up line was effective/humourous, or “bad” if the pick-up line is not satisfactory. Once the user has made a rating, they can see the statistics for that particular pick-up line, including the dominant percentage (e.g. this pick-up line has a 67% “good” rating). The user should then have the option to continue going through more pick-up lines or be prompted to suggest their own. Users should also be able to comment on pick-up lines and converse with other users for interaction with their rating shown once they have placed a rating for it.

Motivation

In a similar vein to 'either.io' (Would You Rather), “Rate My Pick-Up Line” attempts to use a similar rating system and the idea of rating pick-up lines has not been thought up yet in a website concept, thus making a website with a database server that can give users suggestions for what pick-up lines they should use or suggest their own and see how ‘good’ it is. This is also a website that does not take pick-up lines too seriously like ‘Would You Rather’, therefore, this project also aims to entertain users and see what other users come up with, “good” or “bad”.

Tools and Technologies

The tools and technologies required may include, but are not limited to:

  • JavaScript
  • HTML
  • SQL Server
  • Adobe Photoshop

    • With the use of these technologies and tools, we will be able to shape the aesthetic of the website and make sure that data being inputted by users is stored in a SQL Server through JavaScript on the website.

    Skills Required

    The skills that will be required for this project are:

  • Knowledge of SQL
  • UX Design
  • Basic HTML Knowledge
  • JavaScript Knowledge
  • Basic Multi-Media editing skills (photo editing)

    • As the scope of this project should be small, finding the skills required may be easy to do, especially that some of these skills are currently being taught at RMIT.

    Outcome

    If the project becomes successful, then “Rate My Pick-Up Line” should be able to take plenty of user submissions and provide interesting pick-up lines, as well as entertainment for other users looking to interact with the website. The original problem will have been solved by the curation of pick-up lines that the website provides to users that can be rated and then viewed, whether it was “good” or “bad”. The impact of this development is that it will help develop the skills to create a link between a website and a SQL server that is able to serve statistics to the user, as well as provide a proper UX and registration services.
